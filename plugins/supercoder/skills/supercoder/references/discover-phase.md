# Phase 1: Discover

Parse the requirement and build a structured understanding of what the user wants.

## Output

Present naturally — clarity over filling in a form:

- **Core intent** — one sentence
- **Scope** — in / implicit / out
- **Acceptance criteria** — numbered, testable
- **Constraints** — technical, business, compatibility, performance
- **Key terms** — ambiguous or domain-specific definitions

## Steps

1. Analyze the requirement across the five output dimensions above.
2. If the requirement has ambiguities that prevent clear scoping, ask via AskUserQuestion. Do not guess.
