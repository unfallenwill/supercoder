# Phase 3: Clarify

Ask targeted technical questions grounded in codebase findings — architectural choices, pattern conflicts, compatibility concerns.

## Output

After getting answers, present:

- **Refined scope** — updated boundaries
- **Refined acceptance criteria** — updated based on answers
- **Key decisions** — each decision, the answer, and its impact

## Steps

1. Generate up to 8 prioritized questions. Priority order: architectural blockers > scope impact > conflicting interpretations > edge cases.
2. Present via AskUserQuestion, up to 4 per batch. Each question: state the ambiguity, reference specific code, offer 2–3 concrete options.
3. Incorporate answers into refined scope and acceptance criteria.
