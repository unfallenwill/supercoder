# Phase 5: Implement

Write working code, one increment at a time.

## Output

After all increments:

- Files changed and what was done
- Design approach implemented
- Deviations from design (if any)
- Test status

## Increment Principles

- 1–3 files per increment, sequentially by dependency
- Core path before edge cases
- Each increment addresses one concern

## Steps

1. Break the design into increments. Use TaskCreate for each.
2. Execute each increment: implement, run tests, fix failures (max 2 retries, then escalate to user).
3. After the first increment, pause for user checkpoint via AskUserQuestion.
4. If design doesn't fit code reality, roll back to Phase 4.
